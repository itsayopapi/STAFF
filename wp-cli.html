Getting Started with WP-CLI

What is WP-CLI?

WP-CLI is the official command-line interface for WordPress. It provides a complete alternative to the WordPress admin dashboard, allowing you to manage your WordPress installation through terminal commands.

Key Benefits:

  • Execute commands up to 10x faster than the admin interface
  • Automate repetitive tasks with scripts
  • Manage WordPress when GUI isn't available (headless servers)
  • Integrate with deployment workflows and CI/CD pipelines
  • Perform bulk operations with single commands

Use Cases

Development

Quickly set up local environments, manage plugins/themes, and run tests.

Staging/Production

Deploy updates, manage users, and perform maintenance tasks.

Automation

Create scripts for backups, migrations, and scheduled tasks.

Performance

Execute resource-intensive operations without browser overhead.

Installation

Requirements: PHP 5.6 or later, WordPress 3.7 or later, UNIX-like environment (Linux, macOS, WSL on Windows)

Installation Methods

The recommended installation method for Linux and macOS:

curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar
chmod +x wp-cli.phar
sudo mv wp-cli.phar /usr/local/bin/wp

Verify installation:

wp --info

For Windows, we recommend using Windows Subsystem for Linux (WSL):

  1. Install WSL (Windows Subsystem for Linux) from Microsoft Store
  2. Launch Ubuntu or your preferred Linux distribution
  3. Follow the Linux installation instructions above

Native Windows support is limited. For full functionality, use WSL or a virtual machine.

Run WP-CLI in a Docker container:

docker run -it --rm --volumes-from your-wordpress-container --network container:your-wordpress-container wordpress:cli wp --info

For regular usage, create an alias:

alias wp="docker run -it --rm --volumes-from your-wordpress-container --network container:your-wordpress-container wordpress:cli wp"

Manual installation steps:

  1. Download the latest WP-CLI phar file
  2. Place it in your PATH (e.g., /usr/local/bin)
  3. Make it executable
# Download latest release
wget https://github.com/wp-cli/wp-cli/releases/download/v2.8.1/wp-cli-2.8.1.phar -O wp-cli.phar

# Make executable
chmod +x wp-cli.phar

# Move to PATH
sudo mv wp-cli.phar /usr/local/bin/wp

Updating WP-CLI

To update to the latest version:

wp cli update

Installing Tab Completion

Basic Usage

Command Structure

WP-CLI commands follow this pattern:

wp <command> <subcommand> [--<flag>=<value>] [<arguments>]

Getting Help

To see available commands:

wp help

For help with a specific command:

wp help <command>

Common Flags

Flag Description
--path=<path> Path to WordPress installation
--url=<url> Pretend request came from given URL
--user=<id|login|email> Run command as a specific user
--skip-plugins[=<plugins>] Skip loading all or some plugins
--skip-themes[=<themes>] Skip loading all or some themes
--require=<path> Load PHP file before running command
--debug[=<group>] Show all PHP errors or specific debug group

Environment Configuration

You can configure default parameters in wp-cli.yml:

# Sample wp-cli.yml
path: /var/www/wordpress
disabled_commands:
- db drop
- plugin delete

Advanced Usage

Custom Commands

You can create your own WP-CLI commands by creating a custom PHP class:

Example: Hello World Command

<?php
/**
 * Hello World command for WP-CLI
 */
class Hello_World_Command {
    /**
     * Prints a hello world message
     *
     * ## OPTIONS
     * <name>: The name to greet
     * 
     * ## EXAMPLES
     * wp hello-world John
     */
    public function __invoke( $args ) {
        WP_CLI::success( "Hello, " . $args[0] . "!" );
    }
}

// Register the command
WP_CLI::add_command( 'hello-world', 'Hello_World_Command' );

Save this as a PHP file in your plugin or theme, then require it in your wp-cli.yml:

require:
- wp-content/plugins/my-plugin/wp-cli-commands.php

Bulk Operations

WP-CLI excels at performing bulk operations. For example, to update all plugins except specific ones:

wp plugin update --all --exclude=akismet,hello-dolly

To find and replace content across all posts:

wp search-replace 'old text' 'new text' wp_posts --precise

Using WP-CLI in Scripts

You can chain commands together in bash scripts:

Scripting with WP-CLI

Basic Scripting

WP-CLI can be used in shell scripts to automate complex workflows:

Example: Site Backup Script

#!/bin/bash

# Configuration
SITE_PATH="/var/www/wordpress"
BACKUP_DIR="/backups/wordpress"
DATE=$(date +%Y-%m-%d-%H%M%S)

# Create backup directory if it doesn't exist
mkdir -p "$BACKUP_DIR"

# Change to site directory
cd "$SITE_PATH" || exit

# Export database
wp db export "$BACKUP_DIR/db-backup-$DATE.sql"

# Create archive of WordPress files
tar -czf "$BACKUP_DIR/files-backup-$DATE.tar.gz" .

# Clean up backups older than 30 days
find "$BACKUP_DIR" -type f -mtime +30 -delete

echo "Backup completed: $BACKUP_DIR/files-backup-$DATE.tar.gz"

Using WP-CLI in PHP

You can execute WP-CLI commands from PHP scripts:

Example: PHP Script Using WP-CLI

<?php
/**
 * Execute WP-CLI command from PHP
 */
function execute_wp_cli($command) {
    $output = [];
    $return_var = 0;
    
    // Change to WordPress directory
    chdir('/path/to/wordpress');
    
    // Execute command
    exec("wp $command", $output, $return_var);
    
    if ($return_var !== 0) {
        throw new Exception("WP-CLI command failed: " . implode("\n", $output));
    }
    
    return $output;
}

// Example usage
try {
    $plugins = execute_wp_cli('plugin list --format=json');
    $plugins = json_decode(implode("\n", $plugins), true);
    
    foreach ($plugins as $plugin) {
        if ($plugin['update'] === 'available') {
            execute_wp_cli("plugin update {$plugin['name']}");
            echo "Updated plugin: {$plugin['name']}\n";
        }
    }
} catch (Exception $e) {
    echo "Error: " . $e->getMessage();
}

Troubleshooting

Common Issues

"Error: This does not seem to be a WordPress installation."

Solution: Run the command from your WordPress root directory or specify the path:

wp --path=/path/to/wordpress <command>

"PHP Fatal error: Allowed memory size exhausted"

Solution: Increase PHP memory limit:

wp config set WP_MEMORY_LIMIT 256M --raw

Or run WP-CLI with increased memory:

php -d memory_limit=256M $(which wp) <command>

"Error: Failed to connect to database"

Solution: Verify your database credentials in wp-config.php:

wp config get DB_NAME
wp config get DB_USER
wp config get DB_PASSWORD
wp config get DB_HOST

Debugging

Enable debug mode to get more detailed error information:

wp --debug <command>

Check PHP error logs:

wp eval 'echo ini_get("error_log");'

Security

Important: WP-CLI provides powerful access to your WordPress installation and should be properly secured.

Best Practices

  • Restrict access: Only trusted users should have SSH access to your server
  • Use strong passwords: For both system accounts and WordPress users
  • Disable this page: Remove or password-protect this status page in production
  • Monitor usage: Review command history for suspicious activity
  • Keep updated: Regularly update WP-CLI to the latest version

Securing WP-CLI

Disable dangerous commands in wp-cli.yml:

disabled_commands:
- db drop
- plugin delete
- theme delete
- user delete

Restrict WP-CLI to specific users via sudoers file:

# Allow only specific users to run WP-CLI
username ALL=(ALL) NOPASSWD: /usr/local/bin/wp

Best Practices

General Usage

  • Always test commands in staging before production
  • Use --dry-run for potentially destructive operations
  • Create backups before major changes
  • Document frequently used commands
  • Use version control for custom commands

Performance Tips

  • Use --skip-plugins for faster execution
  • Process large datasets in batches
  • Schedule resource-intensive tasks during low-traffic periods
  • Consider using wp eval-file for complex operations
  • Disable WP_DEBUG in production

Development Workflow

Deployment Workflow

  4. Update plugins/themes

Frequently Asked Questions

Run wp --info to verify installation. You should see information about your PHP, WordPress, and WP-CLI versions.

Some shared hosts support WP-CLI. Check with your hosting provider. If SSH access isn't available, you might be able to use WP-CLI through a PHP script.

Run wp cli update. You can also specify a version with wp cli update --version=2.8.1.

Many WP-CLI commands are irreversible. Always use --dry-run first to test commands, and maintain regular backups.

Use the --path parameter to specify different installations, or create a bash script to loop through sites. Some users create aliases for frequently used sites.
