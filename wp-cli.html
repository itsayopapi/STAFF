What is WP-CLI?

WP-CLI is the official command-line interface for WordPress.

It enables you to interact with your WordPress installation without using a web browser. Developers and system administrators use WP-CLI to:

  • Manage plugins, themes, and users
  • Perform database operations
  • Run custom scripts
  • Automate repetitive tasks
Did you know? WP-CLI can process commands up to 10x faster than the WordPress admin dashboard.

Why Use WP-CLI?

Execute commands in seconds without browser overhead. No more waiting for admin pages to load.

Automation

Script repetitive tasks and integrate with deployment workflows. Perfect for CI/CD pipelines.

Server Management

Manage WordPress when GUI isn't available (like on headless servers or during SSH sessions).

Essential WP-CLI Commands

These are the most frequently used commands:

wp --info

Displays information about your WP-CLI environment including PHP version, WP-CLI version, and WordPress installation details.

wp core version

Shows the currently installed WordPress version.

wp plugin list

Lists all installed plugins with their status (active/inactive) and version.

wp user list

Displays all users registered on your WordPress site.

Advanced Features

Database Management

wp db query "SELECT * FROM wp_posts WHERE post_type='post';"

Run direct SQL queries on your WordPress database.

wp db export backup.sql

Create a complete database backup with one command.

Search & Replace

wp search-replace 'http://old.com' 'http://new.com'

Safely replace URLs in your database (perfect for migrations).

wp search-replace 'foo' 'bar' --dry-run

Test replacements before actually making changes (--dry-run flag).

Troubleshooting

Common Errors

"Error: This does not seem to be a WordPress installation."

Solution: Run the command from your WordPress root directory or use --path=/path/to/wordpress.

"PHP Fatal error: Allowed memory size exhausted"

Solution: Increase PHP memory limit by adding to wp-config.php:

define('WP_MEMORY_LIMIT', '256M');

Debugging Tips

Enable Debug Mode

wp config set WP_DEBUG true --raw

Check PHP Error Log

wp eval 'echo ini_get("error_log");'
