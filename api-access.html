Quick Start Example

Here's how to list all your servers using cURL:

# List all servers
curl -X GET \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: application/json" \
https://api.yourhosting.com/v1/servers

Example response:

{
  "data": [
    {
      "id": "srv-12345",
      "name": "Production Web Server",
      "status": "active",
      "ip_address": "203.0.113.45"
    }
  ]
}

Frequently Asked Questions

How do I authenticate with the API?

Include your API key in the Authorization header of each request: Authorization: Bearer YOUR_API_KEY. You can generate API keys in the API Access section of your dashboard.

What is the rate limit for API requests?

Standard API keys are limited to 5,000 requests per hour. If you need higher limits, please contact our support team to discuss enterprise options.

How can I secure my API keys?

Always keep your API keys secret. Use IP restrictions to limit where keys can be used from, rotate keys regularly, and never commit keys to version control. Consider using environment variables or secret management tools.

What happens if I exceed my rate limit?

Requests beyond your rate limit will receive a 429 Too Many Requests response. The response headers will include information about when the limit resets.

How do webhooks work?

Webhooks allow you to receive real-time notifications about events in your account. You can configure webhooks to send HTTP requests to your server when specific events occur.

Where can I get help with API integration?

Our documentation includes detailed guides and examples. For additional help, you can contact our support team or join our community forum to ask other developers.
